## 总体思路

- app是非常零散的功能
- flow组织也不是很方便
- 脚本/action等等概念还是复杂
- 外部是否可以直接调用构建好的workflow?

## 分析

- 每一个脚本都是一些片断代码
- 是否可以远程开发或者在本地仓库开发
- 把开发分割成一小步骤一小步骤
- 所谓的逻辑这些内容，如果已经可以写片段程序了，也就不是很使用
